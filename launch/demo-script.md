- Screen recording of Mac desktop + picture-in-picture of founder (bottom-right corner)
- Founder narrates casually. Not reading a script — talking to a friend.
- Background music: lo-fi ambient, very low. Fades out during voice interactions.

## Production Notes

### Recording Setup
- **Screen capture:** OBS or ScreenFlow, 60fps, retina resolution
- **Audio:** Record founder voice separately (good mic, quiet room). Layer over screen recording.
- **VII audio:** Capture system audio separately for the agent TTS responses
- **PIP camera:** iPhone or webcam, natural lighting, casual setting

### Key Moments to Nail
1. **0:05** — The moment Safari opens by voice command. This is the "holy shit" moment. Must be smooth, no lag.
2. **0:15** — Vision response. Needs to be fast and accurate. Pre-test this exact scenario.
3. **0:25** — Agent switch. The color transition from blue to pink must be visible and satisfying.
4. **0:35** — Memory recall. It needs to reference the earlier flight search naturally. This proves context.
5. **0:45** — Phone control. The Telegram screenshot must be crisp and clearly show the Mac desktop.

### What to Rehearse
- Run through the exact script 3x before recording
- Pre-load Safari with no tabs (clean state)
- Close all notifications
- Set Do Not Disturb
- Test each VII interaction individually first
- Have a backup take for each segment — edit the best together

### Things to Avoid
- Don't show any errors, loading screens, or terminal windows
- Don't show the installation process
- Don't explain the tech — just show the magic
- Don't use a robotic "demo voice" — be natural, slightly excited
- Don't rush — let each moment breathe for half a second

### Music
- Track: Any royalty-free lo-fi ambient (Epidemic Sound or similar)
- Volume: 15-20% during narration, 40% during transitions
- No lyrics
